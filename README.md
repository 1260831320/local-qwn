# 鸿栀 v1.2.0

鸿栀是一套本地优先的智能工作台。

当前版本把重点放在四件事上：

- 用中文优先的界面完成阅读、对话和状态查看
- 在普通电脑上保持可用的本地体验
- 让重要变更先预览、后确认
- 对未开放的能力保持明确说明，不假装已经实现

## 你可以做什么

- 阅读当前版本说明，并按章节快速定位内容
- 在工作台中直接提问，查看回答和处理过程
- 根据需要切换回答引擎与回答模式
- 在同一浏览器刷新后，继续恢复最近一次对话
- 在确认后应用变更，并回看最近记录与详情

## 三个主要页面

### 总览

- 快速了解当前版本能做什么
- 用更直观的流程卡片查看推荐使用顺序
- 了解不同使用模式的特点

### 工作台

- 优先展示当前对话
- 在同一页查看服务状态与使用偏好
- 同一浏览器刷新后，可恢复最近对话和待确认变更
- 遇到变更时先看预览，再决定是否应用
- 已应用或失败的变更都可以在同一处回看

### 手册

- 支持中文与 English 双语阅读
- 支持章节摘要、目录定位和阅读进度
- 看完可以直接带着问题回到工作台

## 使用方式

1. 启动应用。

Windows:

```powershell
.\mvnw.cmd spring-boot:run
```

macOS / Linux:

```bash
./mvnw spring-boot:run
```

2. 在浏览器打开 `http://localhost:8080`
3. 先看“总览”或“手册”，再进入“工作台”
4. 如需应用变更，请先阅读预览内容

## 当前版本说明

- 版本：`v1.2.0`
- 形态：本地优先智能工作台
- 重点：会话恢复、变更历史回看、核心功能优先展示、双语手册阅读

## 本地体验原则

- 本地优先：优先使用本地服务与本地推理
- 隐私友好：不依赖云端必需能力
- 清晰可见：重要状态和当前模式直接展示
- 安全确认：涉及写入时先预览、再确认

## 可用引擎

当前版本提供两种本地回答引擎：

- `Ollama`
- `OpenVINO`

一般建议：

- 问题更复杂、希望回答更完整时，优先使用 `Ollama`
- 更看重轻量运行和资源占用时，可以尝试 `OpenVINO`

## 当前暂未开放

以下区域仍是预留位，当前版本不会伪造能力：

- 语音模式
- 麦克风 / 摄像头连接
- 数字形象表达
- 情绪建议与干预结果

## v1.2.0 更新

- 工作台支持在同一浏览器刷新后恢复最近对话
- 待确认变更会随当前会话一并恢复
- 最近变更会按时间保留，并支持在工作台回看详情
- 回答偏好会更贴近当前本地工作台状态
- 核心内容优先展示与双语手册阅读继续保留为默认体验

## 提醒

- 当前版本仍在持续完善中
- 如果某项服务暂时不可用，界面会直接提示当前状态
- 在变更确认前，不会直接写入文件
